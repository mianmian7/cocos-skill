# Cocos Skill Run Ledger

Use this file to record one live-editor task from baseline through final decision.

## When To Use

Use the ledger when a task has multiple live attempts, non-trivial verification, or a real keep or discard decision.

Do not use the ledger for:

- one-shot reads
- trivial edits with obvious readback
- long-term reusable lessons that belong in `Project Notes`

## How To Use

1. Read `PROGRAM.md` first.
2. Copy or update the sections inside the preserved user block below.
3. Record each attempt with enough evidence for the next agent to understand what happened.
4. End with one decision summary: `keep`, `discard`, or `manual-review`.

## Required Fields

Each run should cover:

- baseline
- live target
- action
- verification
- decision
- evidence

## Decision Rules

- `keep`: verified live state matches the intended outcome
- `discard`: verified live state does not match, or the attempt should not be reused
- `manual-review`: evidence is incomplete, conflicting, or risky enough to stop automation
